# Retrospective — Pong (eerste speelbare release 0.2.0) (2026-06-02)

## Context

Pong is project #1 van P-1: de eerste eindige, echte werklast die de volledige Spec A-route (`/start-project` → `/auto` → promote) doorliep. Eén browser-speelbare, moderne Pong (1P-vs-AI + 2P-lokaal) met vier verplichte uitbreidingen (geluid, AI-graden, pauze/herstart, instelbare matchlengte + visuele juice). Dubbel doel: een goed product én de route uitoefenen onder realistische werklast, met frictie terug naar `/evolve`.

Gebouwd als vanilla TypeScript + Canvas 2D (geen engine), met strikte scheiding tussen pure simulatie (`src/sim/*`) en de IO-schil (render/input/audio/loop). Gepromoot naar main (698bbef, 0.2.0) en gepusht naar de publieke repo.

## Wat werkte

- **Pure/IO-scheiding maakte de happy-path unit-testbaar.** Ball, paddle, collision, score, ai, step, effects en de scene-state-machine zijn deterministische pure functies (geïnjecteerde `dt` + seeded RNG) → 67 vitest-tests dekken de kern-logica zonder DOM/Canvas.
- **Procedurele WebAudio** (geen externe assets) hield de repo self-contained.
- **De adversariële review (gate 4b) vond échte game-bugs**, geen ruis: continue screenshake op elke muur-bounce, dubbele `serveBall` + RNG-drift bij score, dode pos-override, en Enter-start-op-elke-menu-rij. Allemaal gefixt met regressie-tests.
- **Browser-acceptatie (Playwright)** bevestigde de kern-happy-path die unit-tests niet kunnen: laden → menu → match-start → W/S-input → pauze/resume → terug-naar-menu, met lopende animatie en geen runtime-errors.

## Wat anders

- **Browser-acceptatie hoort eerder in de lus.** De eerste promote-poging had alle 13 acceptatiecriteria onaangevinkt; de game was nooit in een browser gedraaid. De adversariële verifier ving dat (missing-evidence). Volgende keer: bouw + browser-smoke als onderdeel van de implementatie, niet pas bij promote.
- **Geen build-script.** De game heeft `dist/` via ad-hoc `esbuild`; een `build`-script in package.json zou de "hoe draai ik dit"-stap stollen.

## Bugs/risico's gevonden

- **Screenshake op wall-bounce** (effects.ts): continu schudden tijdens normaal spel — spec wilde shake alleen bij paddle-hit/score. Gefixt + getest.
- **Niet-deterministische serve** (step.ts): premature `serveBall` bij score verschoof de RNG-sequentie afhankelijk van delay-ticks. Nu precies één serve per punt; bal rust RNG-vrij in center tijdens de delay.
- **Enter startte de match op elke menu-rij** (machine.ts) — instellingen wijzigen met Enter lanceerde het spel. Nu alleen op de START-rij.
- **Pause+confirm in dezelfde tick** → onbedoelde restart i.p.v. resume. Resume krijgt nu voorrang.
